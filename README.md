# тгк без блокировок - База Знаний с Аннотациями

Веб-приложение для создания персональной базы знаний с интерактивными аннотациями, аналог Genius и Wikipedia.

## Возможности

### Основные функции

1. **Регистрация и авторизация**
   - Регистрация с уникальным username (4+ символа, латиница, цифры, _)
   - Вход по email/паролю
   - Первый зарегистрировавшийся пользователь получает права администратора

2. **Лента постов** - главная страница с карточками постов
   - Посты могут содержать несколько типов контента одновременно
   - Текстовые посты с интерактивными аннотациями
   - Фото с возможностью увеличения и полноэкранного просмотра
   - Аудио/видео посты с медиаплеером
   - Файлы для скачивания (любые форматы)
   - Опциональное описание для всех типов постов
   - При клике на выделенный термин открывается всплывающее окно с описанием

3. **Единые аннотации** - каждый термин имеет одну общую страницу
   - Все посты ссылаются на одну версию аннотации
   - При редактировании аннотации обновляется везде

4. **Фильтрация и организация**
   - Фильтрация по типу контента (текст, аудио, видео, фото, файл)
   - Фильтрация по хэштегам
   - Фильтрация по связанным персонам
   - Фильтрация по дате (календарь/хронология)

5. **Админ-панель** (доступна только первому пользователю)
   - Создание и редактирование постов
   - Множественный выбор типов контента для одного поста
   - Загрузка медиафайлов (до 5 ГБ на файл на платных планах Supabase)
   - Поддержка форматов:
     - Аудио: MP3, WAV, OGG, M4A, AAC, WebM
     - Видео: MP4, WebM, OGG, MOV, AVI, MKV
     - Фото: JPG, PNG, GIF, WebP, SVG
     - Файлы: любые форматы
   - Автоматическое распознавание аннотаций в тексте
   - Опциональное описание с галочкой
   - Управление хэштегами и персонами
   - Управление всеми аннотациями

6. **Дополнительные функции**
   - Кнопка "Случайная аннотация"
   - Граф связей аннотаций (соединены аннотации из одного поста)
   - Полноэкранный просмотр фото и видео
   - Zoom для фотографий
   - Скачивание файлов
   - Переключение светлой/темной темы
   - Адаптивный дизайн для смартфонов и планшетов
   - Красивый дизайн с шрифтом Inter

## Технологии

- **Frontend**: React + TypeScript + Vite
- **Styling**: Tailwind CSS
- **Database**: Supabase (PostgreSQL)
- **Storage**: Supabase Storage (для медиафайлов)
- **Authentication**: Supabase Auth
- **Icons**: Lucide React

## Настройка

### 1. База данных уже настроена

База данных Supabase уже создана и настроена. Все необходимые таблицы и политики безопасности развернуты.

### 2. Первая регистрация

Первый зарегистрировавшийся пользователь автоматически получает права администратора.

1. Откройте приложение
2. Нажмите на иконку входа в правом верхнем углу
3. Перейдите на вкладку "Регистрация"
4. Заполните форму:
   - **Username**: минимум 4 символа, только латиница, цифры и подчеркивания
   - **Email**: ваш email
   - **Пароль**: минимум 6 символов
   - **Подтвердите пароль**: введите пароль еще раз
5. Нажмите "Зарегистрироваться"
6. После успешной регистрации войдите с созданными учетными данными

**Важно:** Только первый пользователь получает права администратора. Остальные пользователи могут только просматривать контент.

### 3. Доступ к админ-панели

После входа администратор увидит:
- Бейдж "admin" рядом с username в хедере
- Кнопку "Админ-панель" в правом нижнем углу экрана
- Нажмите на кнопку для перехода к управлению контентом

## Использование

### Создание постов

1. Откройте админ-панель
2. Нажмите вкладку "Управление постами"
3. Заполните форму создания поста:
   - **Название**: введите название поста
   - **Типы контента**: выберите один или несколько типов (можно комбинировать)
   - **Текст**: если выбран тип "Текст", введите текстовое содержание
   - **Файлы**: загрузите файлы для выбранных типов контента
     - Фото: JPG, PNG, GIF, WebP, SVG
     - Аудио: MP3, WAV, OGG, M4A, AAC, WebM
     - Видео: MP4, WebM, OGG, MOV, AVI, MKV
     - Файл: любые форматы для скачивания
   - **Описание**: поставьте галочку и введите описание (необязательно)
   - **Хэштеги**: через запятую (например: наука, история, философия)
   - **Персоны**: через запятую (например: Иван Иванов, Петр Петров)
4. Нажмите "Создать"

**Особенности:**
- Аннотации автоматически определяются в тексте и описании
- Максимальный размер файла: до 5 ГБ (на платных планах Supabase)
- Можно создавать посты с несколькими типами контента одновременно
- Загруженные файлы хранятся в Supabase Storage

### Создание аннотаций

1. Перейдите на вкладку "Управление аннотациями"
2. Введите термин и описание
3. Нажмите "Создать аннотацию"

Аннотации можно создавать заранее или они будут созданы автоматически при добавлении поста.

### Просмотр постов

**Текстовые посты:**
- Термины с аннотациями подсвечены желтым
- Клик на термин открывает всплывающее окно с описанием

**Фото:**
- Клик на фото открывает полноэкранный режим
- В полноэкранном режиме можно увеличивать (zoom) и перемещать фото
- Кнопка скачивания доступна при наведении

**Видео:**
- Встроенный плеер с контролем воспроизведения
- Регулировка скорости воспроизведения
- Полноэкранный режим

**Аудио:**
- Встроенный плеер с визуализацией прогресса
- Регулировка скорости и громкости

**Файлы:**
- Клик на карточку файла скачивает его

**Описание:**
- Если у поста есть описание, его можно развернуть нажатием на "Описание"

### Фильтрация постов

На главной странице:

1. Нажмите "Фильтры"
2. Выберите нужные критерии:
   - Тип контента (текст, аудио, видео, фото, файл)
   - Хэштег
   - Персона
   - Диапазон дат
3. Нажмите "Сбросить" для очистки фильтров

**Примечание:** Посты с несколькими типами контента отображаются при выборе любого из их типов.

### Граф связей

1. Нажмите иконку графа в хедере
2. Увидите все аннотации с количеством связей
3. Клик на аннотацию откроет её описание

### Случайная аннотация

Нажмите кнопку "Случайная" в хедере для открытия случайной аннотации.

## Структура базы данных

### Таблицы

- `posts` - посты (текст, аудио, видео)
- `annotations` - аннотации (термины)
- `post_annotations` - связь между постами и аннотациями
- `hashtags` - хэштеги
- `post_hashtags` - связь между постами и хэштегами
- `persons` - персоны
- `post_persons` - связь между постами и персонами

### Storage

- `media-files` - хранилище для медиафайлов (аудио, видео, фото, файлы)
  - Публичный доступ для чтения
  - Загрузка доступна только администраторам
  - Лимит: до 5 ГБ на файл (на платных планах Supabase)
  - Папки: `audio/`, `video/`, `photo/`, `file/`

## Безопасность

- Все данные доступны для чтения всем пользователям (публичная база знаний)
- Создавать и редактировать контент могут только администраторы (первый пользователь)
- Row Level Security (RLS) настроены для всех таблиц и Storage
- Загружать медиафайлы могут только администраторы
- Загруженные файлы доступны для публичного просмотра и скачивания
- Пароли хранятся в зашифрованном виде в Supabase
- Username уникальный и валидируется на уровне базы данных
- Первый пользователь автоматически получает права администратора
